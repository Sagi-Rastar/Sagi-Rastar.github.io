1 博客是拿来记的

眼睛好干（

博客也写了有一阵子了，个站也逐渐有个样子了。

这几天都想着装修，这里的样式该怎么修改，那里的动画效果能不能做的更好，写出来了想要实现的效果之后，换个设备和浏览器又发现有不适配的地方。文字出框了，图片长宽比不对了，总之 css 和 js 真的是深坑啊——

但是回想起来想要写博客的初衷，还是以记录和积累为主，过多去纠结样式和呈现效果，花这么多时间放在测试代码上面，反而跟一开始的目的背道而驰了。就比如今天下午五点开始，本来打算更新一篇复习用的博客，就收工吃饭，晚上再把作业写了的，然后上传之后发现 Latex 语法没有做渲染适配。

那就适配一下吧，反正才五点。

配置完了，Latex 语法正常渲染了，再翻翻博客，卧槽，我段落之间的段间隔呢，怎么跟行间隔一样。于是又开始了 debug。

是 hexo 的 md 渲染器的问题？好像是，可是 breaks: true 我写在 config 里了呀。

去插件仓库的 issue 里看看？好像有人跟我一样的问题，但是作者在回答中并没有成功复现，给 close 了。

换回我原来的渲染器试试？卧槽。还是没有段间隔。

再换第三个渲染器？卧槽。你怎么也没有。

是 latex 渲染插件的问题？关掉，卧槽。还是没有。

看表，卧槽，外卖到宿舍半个小时了。

再根据 hexo 生成出来的 html 反推，找 hexo 生成文章内容的 ejs，卧槽，看不懂。

万念俱灰了，抬手看表，快八点了。算了再找十分钟的吧，试试直接改 css。

然后我打开了 custom. Css：

嗯？怎么我这里面写过东西的，写了啥？看了一眼注释：

// 段前间距
// 段后间距
“0 px”

啧。

于是这就是我今天三个小时想要找的东西，三个字符让我试了三个 md 渲染器，两个 latex 引擎，以及翻了一堆 ejs。好在找到了之前改的东西，但是想到八点回到宿舍吃完饭之后，又没有那个心情去实验室自习了，就越想越觉得不划算。

以后还是多把时间放在写博客上面吧。

2 真的不看看芙莉莲吗

收敛、克制、平静、宁静、震撼。

关于芙莉莲的评价里被提到了很多次的词，像这样的还有好多。说实话，要是要我来说也肯定会用上这些词。

但是就像芙莉莲这部作品一样，用其中某一次的无聊的小事去描述的话，很难写出这么棒的故事。同样的，不管是芙莉莲动画还是漫画，用某一个准确的词去准确概括他带给我的感受，也太困难了。

我是先看的动画，一口气看了九话之后，精美的制作、恰到好处的音效和配乐、平静但节奏精准地给予观众震撼的剧情，这些都一起给我带来了前所未有的体验。

在旅途中，偶尔克制的剧情流露出的情绪，总能让我鼻头一酸；在战斗的部分，神乎其技的画面和配乐，一起完美的调动着我观看过程中的情绪；在日常欢乐的部分，又总在他该出现的地方给到观众放松；在留白的那些表现上，又将整部作品的基调放到了收敛的基调上，就好像在为下一次的情绪流露做准备。

呃啊，想要表达出来当时的观影体验真的很难，总之大伙都记得看！动画和漫画都推荐！
